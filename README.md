# cafe-audit-dashboard
Cafe Operations Audit Dashboard with interactive questionnaire, risk scoring, and visual analytics including radar, donut, and bar charts.
This project is developed as part of the course **“Auditing IT Infrastructure for IT Compliance.”** It is a web-based application designed to perform an operational and security audit for a cafe environment. The system evaluates key functional areas such as POS and billing controls, kitchen operations, payment handling, inventory tracking, customer data management, and physical security. Based on the responses provided by the user, the application assesses risks, calculates a compliance score, and visualizes audit results using an interactive dashboard.

🎯 **Objectives**
The objective of this project is to simulate a real-world cafe audit process and evaluate operational efficiency along with system security. It aims to identify potential risks in billing systems, payment processes, inventory control, and customer data handling. The system also focuses on providing a structured compliance evaluation and presenting the audit results in a clear and visual format for better understanding and decision-making.

⚙️ **Features**
The application includes an interactive audit questionnaire with Yes, Partial, and No response options. It implements domain-based auditing covering POS and billing, kitchen operations, payment controls, inventory control, customer data, and physical security. A weighted risk scoring system calculates a score ranging from 0 to 100. Based on the score, the system classifies risk levels such as Low, Medium, and High. The dashboard dynamically displays results using donut charts, radar charts, and bar charts. It also provides domain-wise performance analysis, findings by severity, and findings by domain visualization, which update automatically after generating the audit.

🧠 **Concepts Implemented**
This project demonstrates IT and operational auditing techniques, risk assessment and scoring, compliance evaluation, domain-based audit analysis, and interactive data visualization. It also highlights how web-based dashboards can support audit decision-making.

🛠️ **Tech Stack**
Frontend: HTML, CSS, JavaScript
Visualization: Chart.js
Platform: Web-based application

📊 **Workflow**
The user answers the audit questionnaire, the system calculates the risk score, determines the risk level, generates domain scores, and visualizes the results through interactive charts.
